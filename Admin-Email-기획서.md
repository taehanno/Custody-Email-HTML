## 1. 이메일 발송 원칙

| 항목 | 원칙 |
| --- | --- |
| 발송 최소화 | Admin 내부 직원 대상 — 꼭 필요한 이벤트에만 발송 |
| 통일된 포맷 | Front 이메일과 동일한 헤더·푸터 구조 사용 |
| 핵심 정보 우선 | 누가, 언제, 무슨 내용인지 3줄 이내로 즉시 파악 |
| 피싱 주의 문구 | 모든 이메일 하단에 보안 경고 문구 고정 삽입 |
| 발송 시각 표기 | 이벤트 발생 시각을 YY-MM-DD hh:mm (KST) 형식으로 표기 |
| 보안 이벤트 우선 | 본인 계정 침해·탈취 감지 이벤트는 반드시 발송 |
| 결재 분리 원칙 | AML 기안(담당자) ↔ 결재(관리자) 분리 — 각 단계 담당자에게만 필요 시점 발송 |

> 공통 이메일 레이아웃 및 보안 안내 푸터는 Front 이메일 발송 기획서 섹션 2 참조.
